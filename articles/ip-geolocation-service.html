Deploy and Publish Your Own API Wrapper for an IP Geolocation Service

In today's interconnected digital landscape, knowing the geographical location of your website visitors is no longer a luxury—it's a necessity. Whether you're personalizing content, enforcing region-based access controls, analyzing traffic patterns, or combating fraud, an IP geolocation service provides the raw data you need. However, directly integrating with multiple third-party APIs can quickly become messy, leading to code duplication, rate-limit headaches, and vendor lock-in. The solution? Build and deploy your own API wrapper. This guide walks you through the entire process—from design to deployment—so you can publish a clean, reusable, and scalable wrapper for any IP geolocation service.

Why Build an API Wrapper for IP Geolocation?

Before diving into the code, it's important to understand the "why." An API wrapper acts as an intermediary layer between your application and the external IP geolocation service. Instead of calling the provider's API directly from every endpoint in your app, you centralize those calls into a single, well-documented module. This approach offers several key benefits:

  • Abstraction: Your application code never needs to know which geolocation provider you're using. Swap providers with minimal changes.
  • Consistency: Normalize response formats across different providers (e.g., ipapi, ipinfo.io, MaxMind).
  • Resilience: Implement caching, retries, and fallback providers in one place.
  • Maintainability: Update API keys, endpoints, or authentication methods without touching your core business logic.

By the end of this article, you'll have a production-ready wrapper that you can deploy as a microservice or publish as a package for other developers to use.

Planning Your API Wrapper Architecture

A solid wrapper starts with a clean design. Here's what you need to consider:

1. Choose Your Core Provider(s)

Start with one primary IP geolocation service like ipapi (free tier available) or ipinfo.io. Later, you can add fallback providers. For this tutorial, we'll use ipapi as the default.

2. Define a Unified Response Schema

Every provider returns data in a slightly different format. Your wrapper should normalize these into a consistent JSON structure. For example:

{
  "ip": "8.8.8.8",
  "city": "Mountain View",
  "region": "California",
  "country": "US",
  "latitude": 37.386,
  "longitude": -122.0838,
  "isp": "Google LLC",
  "timezone": "America/Los_Angeles"
}

3. Plan for Caching and Rate Limits

Most IP geolocation services impose rate limits, especially on free tiers. Your wrapper should include an in-memory cache (e.g., with TTL) to avoid redundant API calls. A simple dictionary with expiration times works for small-scale apps; for production, use Redis.

Step-by-Step Implementation (Python + Flask)

We'll build the wrapper as a lightweight Flask microservice. This makes it easy to deploy as a standalone API that any application can consume.

Step 2: Create the Core Wrapper Module

Create a file called geo_wrapper.py. This will contain the logic for calling the IP geolocation service and normalizing the response.

import os
import requests
from datetime import datetime, timedelta

class IPGeoWrapper:
    def __init__(self, api_key=None, provider='ipapi', cache_ttl=300):
        self.api_key = api_key or os.getenv('IPAPI_KEY')
        self.provider = provider
        self.cache = {}
        self.cache_ttl = cache_ttl

    def _get_from_cache(self, ip):
        if ip in self.cache:
            entry = self.cache[ip]
            if datetime.now() < entry['expires']:
                return entry['data']
        return None

    def _set_cache(self, ip, data):
        self.cache[ip] = {
            'data': data,
            'expires': datetime.now() + timedelta(seconds=self.cache_ttl)
        }

    def lookup(self, ip):
        # Check cache first
        cached = self._get_from_cache(ip)
        if cached:
            return cached

        # Call the IP geolocation service
        if self.provider == 'ipapi':
            url = f"http://api.ipapi.com/{ip}?access_key={self.api_key}"
        else:
            raise ValueError(f"Unsupported provider: {self.provider}")

        response = requests.get(url, timeout=5)
        response.raise_for_status()
        raw_data = response.json()

        # Normalize the response
        normalized = {
            'ip': ip,
            'city': raw_data.get('city'),
            'region': raw_data.get('region_name'),
            'country': raw_data.get('country_code'),
            'latitude': raw_data.get('latitude'),
            'longitude': raw_data.get('longitude'),
            'isp': raw_data.get('connection', {}).get('isp'),
            'timezone': raw_data.get('timezone', {}).get('id')
        }

        self._set_cache(ip, normalized)
        return normalized

Step 3: Build the Flask API Endpoint

Now create app.py to expose your wrapper as a RESTful API:

from flask import Flask, request, jsonify
from geo_wrapper import IPGeoWrapper
import os

app = Flask(__name__)
geo = IPGeoWrapper(api_key=os.getenv('IPAPI_KEY'))

@app.route('/api/geo/', methods=['GET'])
def get_geolocation(ip):
    try:
        data = geo.lookup(ip)
        return jsonify(data), 200
    except Exception as e:
        return jsonify({'error': str(e)}), 500

@app.route('/api/geo', methods=['GET'])
def get_own_geolocation():
    # Get the requester's IP
    ip = request.remote_addr
    try:
        data = geo.lookup(ip)
        return jsonify(data), 200
    except Exception as e:
        return jsonify({'error': str(e)}), 500

if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=5000)

Step 4: Add Error Handling and Retries

Production wrappers need resilience. Enhance the lookup method with retries and fallback logic:

import time

def lookup(self, ip, retries=3):
    for attempt in range(retries):
        try:
            # ... existing lookup code ...
            return normalized
        except requests.exceptions.RequestException as e:
            if attempt == retries - 1:
                raise
            time.sleep(2 ** attempt)  # Exponential backoff

Deploying Your API Wrapper

Once your wrapper is working locally, it's time to deploy it so others can use it. Here are three practical deployment options:

Option 1: Deploy as a Microservice on Render

Render offers a free tier perfect for small APIs:

  1. Push your code to a GitHub repository.
  2. On Render, create a new "Web Service" and connect your repo.
  3. Set the start command to gunicorn app:app (install gunicorn via pip install gunicorn).
  4. Add environment variable IPAPI_KEY with your API key.
  5. Deploy! Your IP geolocation service wrapper will be live at a URL like https://your-app.onrender.com.
