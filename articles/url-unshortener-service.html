How to Deploy and Publish a URL Unshortener Service API Wrapper

In the modern web ecosystem, URL shorteners are everywhere—from marketing campaigns to social media posts. While they serve a purpose, they also introduce risks: you never know where a shortened link will take you. That's where a URL unshortener service comes in. By deploying and publishing an API wrapper around such a service, you can empower users, developers, and businesses to safely expand links, check redirect chains, and enhance security. In this guide, we'll walk through the entire process, from planning to deployment, with actionable tips and best practices.

Why Build a URL Unshortener Service API Wrapper?

A URL unshortener service resolves shortened URLs (like bit.ly, t.co, or ow.ly) to their original destinations. An API wrapper simplifies interaction with that service, making it easy to integrate into applications. Here's why you'd want to build one:

  • Security: Users can preview where a link leads before clicking, reducing phishing and malware risks.
  • Transparency: Developers can log or analyze redirect chains for SEO, analytics, or auditing purposes.
  • Automation: Marketing teams can verify campaign links without manual checking.
  • Customization: You can add caching, rate limiting, or additional features on top of the base service.

By publishing your wrapper as an API, you create a reusable tool that others can leverage—whether for personal projects, internal use, or public consumption.

Step 1: Choose Your URL Unshortener Service Backend

Before writing any code, you need a reliable source of truth. Several free and paid URL unshortener services exist, each with different features and limitations:

Popular Options

  • unshorten.me – Free and straightforward, returns the expanded URL and status.
  • CheckShortURL – Provides detailed information including redirect chain and safety ratings.
  • ExpandURL – Simple API with JSON responses, good for quick integrations.
  • VirusTotal – Includes security scanning, ideal for threat detection use cases.

For this tutorial, we'll assume you're using a free service like unshorten.me. However, the same principles apply to any REST-based API.

Step 2: Design Your API Wrapper

A well-designed API wrapper abstracts the underlying service and provides a clean interface. Your wrapper should handle:

  • Input validation: Ensure the URL is valid and properly formatted.
  • Error handling: Manage timeouts, invalid responses, and service outages gracefully.
  • Response normalization: Return consistent JSON regardless of the backend's quirks.
  • Optional caching: Store results to reduce redundant API calls.

Example Endpoint Design

Your API could expose a single endpoint:

GET /api/unshorten?url=https://bit.ly/example

Response:

{
  "original_url": "https://www.example.com/long-path",
  "status": "success",
  "redirect_count": 2,
  "final_url": "https://www.example.com/long-path"
}

Step 3: Build the Wrapper (Python Example)

Let's implement a simple wrapper using Python and Flask. This example uses the requests library to call an external unshortener service.

Prerequisites

  • Python 3.8+
  • Flask for the web framework
  • Requests for HTTP calls

Implementation

from flask import Flask, request, jsonify
import requests
import validators

app = Flask(__name__)

UNSHORTEN_API_URL = "https://api.unshorten.me/v1"

@app.route('/api/unshorten', methods=['GET'])
def unshorten_url():
    url = request.args.get('url')
    
    # Validate input
    if not url or not validators.url(url):
        return jsonify({"error": "Invalid URL provided"}), 400
    
    try:
        # Call the external service
        response = requests.get(f"{UNSHORTEN_API_URL}?url={url}", timeout=10)
        response.raise_for_status()
        data = response.json()
        
        # Normalize response
        return jsonify({
            "original_url": url,
            "final_url": data.get("destination", url),
            "redirect_count": data.get("redirect_count", 0),
            "status": "success"
        })
    except requests.exceptions.Timeout:
        return jsonify({"error": "Request timed out"}), 504
    except requests.exceptions.RequestException as e:
        return jsonify({"error": str(e)}), 502

if __name__ == '__main__':
    app.run(debug=True)

Key Features

  • Input validation: Uses the validators library to reject malformed URLs.
  • Timeout handling: Prevents your API from hanging if the backend is slow.
  • Consistent output: Always returns JSON with clear status fields.

Step 4: Add Caching for Performance

Without caching, every request hits the external service, which can be slow and expensive. Implement a simple in-memory cache (or use Redis for production):

from functools import lru_cache

@lru_cache(maxsize=100)
def fetch_unshortened(url):
    # Your external API call logic here
    pass

For production, consider using Redis with TTL (time-to-live) to avoid stale data:

import redis
r = redis.Redis(host='localhost', port=6379, db=0)

def get_cached_or_fetch(url):
    cached = r.get(url)
    if cached:
        return cached.decode()
    result = fetch_unshortened(url)
    r.setex(url, 3600, result)  # Cache for 1 hour
    return result

Step 5: Deploy Your API Wrapper

Now that your wrapper is ready, it's time to deploy it so others can use it. Here are three common approaches:

Option A: Deploy on a Cloud Platform (Heroku, Render, or Railway)

  1. Create a requirements.txt file with your dependencies (Flask, requests, validators, gunicorn).
  2. Add a Procfile for Heroku: web: gunicorn app:app.
  3. Push to Git and connect to your cloud provider.
  4. Set environment variables for API keys (if any).

Option B: Containerize with Docker

Docker ensures your API runs consistently anywhere:

FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["gunicorn", "-b", "0.0.0.0:5000", "app:app"]

Then build and push to Docker Hub or a registry.

Option C: Serverless (AWS Lambda + API Gateway)

For low-traffic scenarios, serverless can be cost-effective. Use the serverless framework with Python:

  • Define your function in handler.py.
  • Configure API Gateway to trigger the Lambda.
  • Set memory and timeout appropriately (10 seconds recommended).

Step 6: Publish and Document Your API

A great API is useless if no one knows how to use it. Follow these best practices:

Create Clear Documentation

  • Endpoint: GET https://your-api.com/api/unshorten
  • Parameters: url (required, string) – the shortened URL to expand.
  • Example request: curl "https://your-api.com/api/unshorten?url
